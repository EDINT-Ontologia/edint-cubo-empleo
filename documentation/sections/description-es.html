Vocabulario para la representación del cubo de indicadores de empleo: Descripción volver a índice

Para la generación del cubo de datos de empleo se reutiliza el vocabulario W3C Data Cube. Este vocabulario es una recomendación del W3C para la publicación de datos multidimensionales en la Web, lo que facilita su representación estructurada, su interoperabilidad y su reutilización. Asimismo, este vocabulario es compatible con el estándar SDMX (Statistical Data and Metadata eXchange), ampliamente utilizado para el intercambio de datos y metadatos estadísticos entre organizaciones.
Como se puede observar en la Figura 1, W3C Data Cube contiene varios conceptos y propiedades que permiten describir datos estadísticos con diferentes niveles de agregación. En este vocabulario, el modelo se organiza alrededor del concepto de qb:DataSet, que representa un conjunto de datos que sigue una estructura definida mediante una qb:DataStructureDefinition y que se materializa a través de un conjunto de observaciones representadas por qb:Observation. Concretamente, este vocabulario define un único conjunto de datos, representado por la instancia edintemp:, y una única estructura de datos, representada por la instancia edintemp:dsd. Esta estructura recoge todas las dimensiones y medidas necesarias para representar distintos indicadores del mercado laboral.
  • qb:DimensionProperty: componente que identifica las dimensiones de las observaciones, como por ejemplo la región administrativa de referencia, el periodo de referencia, el sexo, la edad, la actividad económica o el tipo de contrato. Las dimensiones suelen tomar sus valores de listas de códigos o clasificaciones, normalmente representadas mediante SKOS.
  • qb:MeasureProperty: componente que representa el valor medido de los datos observados. En este cubo, las medidas corresponden a indicadores de empleo como el número de personas activas, ocupadas, desempleadas, demandantes de empleo, contratos registrados, paro registrado o afiliaciones a la Seguridad Social.
Para la definición de estas dimensiones y medidas se reutilizan varios vocabularios de referencia:
  1. Vocabularios de SDMX (Statistical Data and Metadata eXchange):
    • sdmx-dimension: se reutiliza como base para dimensiones estadísticas generales, especialmente a través de relaciones de especialización como edintdimension:refAdminRegion, edintdimension:refPeriod, edintdimension:sex y edintdimension:age.
    • sdmx-measure: se reutiliza como base para las medidas del cubo, todas ellas definidas como subpropiedades de sdmx-measure:obsValue.
  2. Vocabularios territoriales y temporales:
    • esadm: se reutiliza para indicar que la dimensión edintdimension:refAdminRegion puede tomar valores correspondientes a municipios, barrios o distritos.
    • OWL-Time: se reutiliza para modelar la dimensión temporal, estableciendo que edintdimension:refPeriod tiene como rango time:Interval.
  3. Listas de códigos SKOS:
    • La dimensión edintdimension:sector reutiliza como lista de códigos el esquema de sectores del sector público publicado en datos.gob.es.
    • La dimensión edintdimension:economicActivity reutiliza una clasificación controlada de actividad económica, como la CNAE 2025.
    • La dimensión edintdimension:contractType utiliza una lista de códigos propia definida en edintkos:contractType.
El modelo del cubo de empleo se representa en la Figura 2, donde se muestran las dimensiones y medidas que lo componen.
La estructura del cubo de empleo se define mediante la instancia edintemp:dsd, que agrupa todas las dimensiones y medidas del modelo. En este caso, se han definido como dimensiones la región administrativa de referencia, el sector, la actividad económica, el periodo de referencia, el sexo, la edad y el tipo de contrato. Como medidas, se incluyen distintos recuentos asociados al empleo, tales como el número de personas activas, ocupadas, desempleadas, contratos registrados, demandantes de empleo, paro registrado y afiliaciones a la Seguridad Social. 
edintemp: a qb:DataSet ; 
    qb:structure edintemp:dsd .

edintemp:dsd a qb:DataStructureDefinition ;
    qb:component
        [ qb:dimension edintdimension:refAdminRegion ; qb:order 1 ] ,
        [ qb:dimension edintdimension:sector ; qb:order 2 ] ,
        [ qb:dimension edintdimension:economicActivity ; qb:order 3 ] ,
        [ qb:dimension edintdimension:refPeriod ; qb:order 4 ] ,
        [ qb:dimension edintdimension:sex ; qb:order 5 ] ,
        [ qb:dimension edintdimension:age ; qb:order 6 ] ,
        [ qb:dimension edintdimension:contractType ; qb:order 7 ] ,
        [ qb:measure edintmeasure:activePopulationCount ; qb:order 8 ] ,
        [ qb:measure edintmeasure:employedPopulationCount ; qb:order 9 ] ,
        [ qb:measure edintmeasure:unemployedPopulationCount ; qb:order 10 ] ,
        [ qb:measure edintmeasure:registeredContractsCount ; qb:order 11 ] ,
        [ qb:measure edintmeasure:jobSeekersCount ; qb:order 12 ] ,
        [ qb:measure edintmeasure:registeredUnemploymentCount ; qb:order 13 ] ,
        [ qb:measure edintmeasure:socialSecurityAffiliatesCount ; qb:order 14 ] .

Ejemplo 1: Número de ocupados según área, periodo, sexo y grupo de edad volver a índice

En este ejemplo se representan los datos del número de personas ocupadas en una determinada región administrativa, para un periodo de referencia concreto, desagregados por sexo y grupo de edad. Dado que en este vocabulario ya están definidos el conjunto de datos edintemp: y la estructura de datos edintemp:dsd, no es necesario generar nuevas instancias de dataset o de estructura, sino reutilizar las existentes.
A continuación se describen los pasos necesarios para representar una observación concreta. **1) Reutilización del conjunto de datos y de la estructura de datos:** En este vocabulario existe un único dataset, representado por edintemp:, que se enlaza con la estructura edintemp:dsd mediante la propiedad qb:structure. 
edintemp: a qb:DataSet ; 
    qb:structure edintemp:dsd .
**2) Identificación de las dimensiones y medidas relevantes:** Aunque la estructura edintemp:dsd contiene todas las dimensiones y medidas del cubo de empleo, en este ejemplo solo se utilizan las dimensiones edintdimension:refAdminRegion, edintdimension:refPeriod, edintdimension:sex y edintdimension:age, junto con la medida edintmeasure:employedPopulationCount. Es importante destacar que la dimensión edintdimension:refPeriod tiene como rango time:Interval, por lo que el periodo de referencia debe representarse mediante una instancia explícita de intervalo temporal. 
<http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/interval/2025>
    a time:Interval ;
    rdfs:label "Year 2025"@en ,
               "Año 2025"@es .
**3) Definición de una observación del dataset:** El paso final consiste en crear una instancia de qb:Observation y enlazarla con el conjunto de datos mediante la propiedad qb:dataSet. A continuación, se asignan los valores correspondientes a las dimensiones y a la medida seleccionadas. 
<http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/observation/obs001>
    a qb:Observation ;
    qb:dataSet edintemp: ;
    edintdimension:refAdminRegion <http://vocab.linkeddata.es/datosabiertos/recurso/territorio/municipio/zaragoza> ;
    edintdimension:refPeriod <http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/interval/2025> ;
    edintdimension:sex <http://purl.org/linked-data/sdmx/2009/code#sex-F> ;
    edintdimension:age "25-29"^^xsd:string ;
    edintmeasure:employedPopulationCount "12543"^^xsd:nonNegativeInteger .
En esta observación se indica que, en el municipio de Zaragoza, durante el intervalo temporal correspondiente al año 2025, el número de mujeres ocupadas del grupo de edad de 25 a 29 años es de 12.543. Este ejemplo muestra cómo una única observación del cubo combina varios valores de dimensión con una medida concreta para representar un indicador de empleo. Del mismo modo, la misma estructura puede reutilizarse para representar otras medidas del cubo, como el número de personas activas, desempleadas, contratos registrados, demandantes de empleo, paro registrado o afiliadas a la Seguridad Social.
