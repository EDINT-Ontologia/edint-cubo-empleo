Vocabulario para la representación del cubo de indicadores de empleo: Descripción volver al índice

Para la generación del cubo de datos de empleo, se reutiliza el vocabulario W3C Data Cube. Este vocabulario es una recomendación del W3C para la publicación de datos multidimensionales en la Web, facilitando su representación estructurada, interoperabilidad y reutilización. Además, este vocabulario es compatible con el estándar SDMX (Statistical Data and Metadata eXchange), ampliamente utilizado para el intercambio de datos y metadatos estadísticos entre organizaciones.
Como se muestra en la Figura 1, W3C Data Cube contiene varios conceptos y propiedades que permiten describir datos estadísticos en distintos niveles de agregación. En este vocabulario, el modelo se organiza en torno al concepto qb:DataSet, que representa un conjunto de datos definido de acuerdo con una estructura especificada mediante una qb:DataStructureDefinition y materializado a través de un conjunto de observaciones representadas con qb:Observation. En concreto, este vocabulario define un único dataset del cubo de datos, representado mediante la instancia edintemp:dataset, y una única estructura de datos, representada mediante la instancia edintemp:dsd. La ontología se identifica de forma separada del dataset, utilizando la IRI de ontología http://vocab.linkeddata.es/datosabiertos/def/actividad-economica/cubo-empleo. Esta separación permite que los metadatos describan la ontología, mientras que edintemp:dataset representa el dataset del cubo al que se vinculan las observaciones.
  • qb:DimensionProperty: componente que identifica las dimensiones de las observaciones, como la región administrativa de referencia, el periodo de referencia, el sexo, la edad, la actividad económica, el sector o el tipo de contrato. Las dimensiones suelen tomar sus valores de listas de códigos o clasificaciones, normalmente representadas mediante SKOS.
  • qb:MeasureProperty: componente que representa el valor medido de los datos observados. En este cubo, las medidas corresponden a indicadores de empleo, como el número de personas activas, personas ocupadas, personas desempleadas, demandantes de empleo, contratos registrados, paro registrado, afiliaciones a la Seguridad Social, tasa de paro, tasa de empleo, puestos de trabajo, perfiles de empleo y número de personas.
Para definir estas dimensiones y medidas se reutilizan varios vocabularios de referencia:
  1. Vocabularios SDMX (Statistical Data and Metadata eXchange):
    • sdmx-dimension: se reutiliza como base para dimensiones estadísticas generales, especialmente mediante relaciones de especialización como edintdimension:refAdminRegion, edintdimension:refPeriod, edintdimension:sex y edintdimension:age.
    • sdmx-measure: se reutiliza como base para varias medidas del cubo, definidas como subpropiedades de sdmx-measure:obsValue.
  2. Vocabularios territoriales y temporales:
    • esadm: se reutiliza para indicar que la dimensión edintdimension:refAdminRegion puede tomar valores correspondientes a municipios, barrios o distritos.
    • OWL-Time: se reutiliza para modelar la dimensión temporal, estableciendo que edintdimension:refPeriod tiene como rango time:Interval.
  3. Listas de códigos SKOS:
    • La dimensión edintdimension:sector reutiliza como lista de códigos el esquema del sector público publicado en datos.gob.es.
    • La dimensión edintdimension:economicActivity reutiliza una clasificación controlada de actividad económica, como CNAE 2025.
    • La dimensión edintdimension:contractType utiliza una lista de códigos propia definida en edintkos:contractType.
  4. Vocabulario del INE:
    • El vocabulario también reutiliza varias propiedades del vocabulario del INE, como ine:unemploymentRate, ine:employmentRate, ine:jobs e ine:numberOfPeople, con el objetivo de extender el cubo con indicadores adicionales relacionados con el empleo.
El modelo del cubo de empleo se muestra en la Figura 2, donde se presentan las dimensiones y medidas que lo componen.
La estructura del cubo de empleo se define mediante la instancia edintemp:dsd, que agrupa todas las dimensiones y medidas del modelo. En este caso, se han definido como dimensiones la región administrativa de referencia, el sector, la actividad económica, el periodo de referencia, el sexo, la edad y el tipo de contrato. Como medidas, el modelo incluye distintos recuentos e indicadores asociados al empleo, como el número de personas activas, ocupadas y desempleadas, contratos registrados, demandantes de empleo, paro registrado, afiliaciones a la Seguridad Social, tasa de paro, tasa de empleo, puestos de trabajo, perfiles de empleo y número de personas. 
<http://vocab.linkeddata.es/datosabiertos/def/actividad-economica/cubo-empleo>
    a owl:Ontology ;
    owl:imports <http://purl.org/linked-data/cube> ;
    owl:versionInfo "0.1.0" .

edintemp:dataset a qb:DataSet ; 
    qb:structure edintemp:dsd .

edintemp:dsd a qb:DataStructureDefinition ;
    qb:component
        [ qb:dimension edintdimension:refAdminRegion ; qb:order 1 ] ,
        [ qb:dimension edintdimension:sector ; qb:order 2 ] ,
        [ qb:dimension edintdimension:economicActivity ; qb:order 3 ] ,
        [ qb:dimension edintdimension:refPeriod ; qb:order 4 ] ,
        [ qb:dimension edintdimension:sex ; qb:order 5 ] ,
        [ qb:dimension edintdimension:age ; qb:order 6 ] ,
        [ qb:dimension edintdimension:contractType ; qb:order 7 ] ,
        [ qb:measure edintmeasure:activePopulationCount ; qb:order 8 ] ,
        [ qb:measure edintmeasure:employedPopulationCount ; qb:order 9 ] ,
        [ qb:measure edintmeasure:unemployedPopulationCount ; qb:order 10 ] ,
        [ qb:measure edintmeasure:registeredContractsCount ; qb:order 11 ] ,
        [ qb:measure edintmeasure:jobSeekersCount ; qb:order 12 ] ,
        [ qb:measure edintmeasure:registeredUnemploymentCount ; qb:order 13 ] ,

        [ qb:measure ine:unemploymentRate ; qb:order 15 ] ,

        [ qb:measure ine:jobs ; qb:order 17 ] ,
        [ qb:measure edintmeasure:jobProfiles ; qb:order 18 ] ,
        [ qb:measure ine:numberOfPeople ; qb:order 19 ] .

Ejemplo 1: Número de personas ocupadas por área, periodo, sexo y grupo de edad volver al índice

Este ejemplo representa el dato correspondiente al número de personas ocupadas en una región administrativa determinada, para un periodo de referencia específico, desglosado por sexo y grupo de edad. Dado que este vocabulario ya define el dataset edintemp:dataset y la estructura de datos edintemp:dsd, no es necesario generar nuevas instancias de dataset o de estructura; se pueden reutilizar las ya existentes.
A continuación, se describen los pasos necesarios para representar una observación concreta. 1) Reutilización del dataset y de la estructura de datos: Este vocabulario contiene un único dataset, representado por edintemp:dataset, que se vincula con la estructura edintemp:dsd mediante la propiedad qb:structure. 
edintemp:dataset a qb:DataSet ; 
    qb:structure edintemp:dsd .
2) Identificación de las dimensiones y medidas relevantes: Aunque la estructura edintemp:dsd contiene todas las dimensiones y medidas del cubo de empleo, este ejemplo solo utiliza las dimensiones edintdimension:refAdminRegion, edintdimension:refPeriod, edintdimension:sex y edintdimension:age, junto con la medida edintmeasure:employedPopulationCount. Es importante señalar que la dimensión edintdimension:refPeriod tiene como rango time:Interval, por lo que el periodo de referencia debe representarse mediante una instancia explícita de intervalo temporal. 
<http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/interval/2025>
    a time:Interval ;
    rdfs:label "Year 2025"@en ,
               "Año 2025"@es .
3) Definición de una observación del dataset: El último paso consiste en crear una instancia de qb:Observation y vincularla al dataset mediante la propiedad qb:dataSet. A continuación, se asignan los valores correspondientes para las dimensiones y la medida seleccionadas. 
<http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/observation/obs001>
    a qb:Observation ;
    qb:dataSet edintemp:dataset ;
    edintdimension:refAdminRegion <http://vocab.linkeddata.es/datosabiertos/recurso/territorio/municipio/zaragoza> ;
    edintdimension:refPeriod <http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/interval/2025> ;
    edintdimension:sex <http://purl.org/linked-data/sdmx/2009/code#sex-F> ;
    edintdimension:age "25-29"^^xsd:string ;
    edintmeasure:employedPopulationCount "12543"^^xsd:nonNegativeInteger .
Esta observación indica que, en el municipio de Zaragoza, durante el intervalo temporal correspondiente al año 2025, el número de mujeres ocupadas en el grupo de edad de 25 a 29 años es de 12.543. Este ejemplo muestra cómo una única observación del cubo combina varios valores de dimensiones con una medida concreta para representar un indicador de empleo. De la misma manera, la misma estructura puede reutilizarse para representar otras medidas del cubo, como el número de personas activas, desempleadas, contratos registrados, demandantes de empleo, paro registrado, afiliaciones a la Seguridad Social, tasa de paro, tasa de empleo, puestos de trabajo, perfiles de empleo o número de personas.
