Vocabulary for the representation of the employment indicators data cube: Description back to index

For the generation of the employment data cube, the W3C Data Cube vocabulary is reused. This vocabulary is a W3C recommendation for the publication of multidimensional data on the Web, which facilitates their structured representation, interoperability, and reuse. In addition, this vocabulary is compatible with the SDMX standard (Statistical Data and Metadata eXchange), which is widely used for the exchange of statistical data and metadata across organizations.
As shown in Figure 1, W3C Data Cube contains several concepts and properties that make it possible to describe statistical data at different levels of aggregation. In this vocabulary, the model is organized around the concept of qb:DataSet, which represents a dataset following a structure defined by a qb:DataStructureDefinition and materialized through a set of observations represented by qb:Observation. Specifically, this vocabulary defines a single dataset, represented by the instance edintemp:, and a single data structure, represented by the instance edintemp:dsd. This structure gathers all the dimensions and measures required to represent different labor market indicators.
  • qb:DimensionProperty: component that identifies the dimensions of the observations, such as the reference administrative region, reference period, sex, age, economic activity, or contract type. Dimensions usually take their values from code lists or classifications, typically represented using SKOS.
  • qb:MeasureProperty: component that represents the measured value of the observed data. In this cube, the measures correspond to employment indicators such as the number of active persons, employed persons, unemployed persons, job seekers, registered contracts, registered unemployment, or Social Security affiliates.
Several reference vocabularies are reused to define these dimensions and measures:
  1. SDMX vocabularies (Statistical Data and Metadata eXchange):
    • sdmx-dimension: reused as the basis for general statistical dimensions, especially through specialization relationships such as edintdimension:refAdminRegion, edintdimension:refPeriod, edintdimension:sex and edintdimension:age.
    • sdmx-measure: reused as the basis for the cube measures, all of them defined as subproperties of sdmx-measure:obsValue.
  2. Territorial and temporal vocabularies:
    • esadm: reused to indicate that the dimension edintdimension:refAdminRegion may take values corresponding to municipalities, neighborhoods, or districts.
    • OWL-Time: reused to model the temporal dimension, establishing that edintdimension:refPeriod has time:Interval as its range.
  3. SKOS code lists:
    • The dimension edintdimension:sector reuses as a code list the public sector scheme published at datos.gob.es.
    • The dimension edintdimension:economicActivity reuses a controlled classification of economic activity, such as CNAE 2025.
    • The dimension edintdimension:contractType uses its own code list defined in edintkos:contractType.
The employment cube model is shown in Figure 2, where the dimensions and measures that compose it are presented.
The structure of the employment cube is defined through the instance edintemp:dsd, which groups all the dimensions and measures of the model. In this case, the reference administrative region, sector, economic activity, reference period, sex, age, and contract type have been defined as dimensions. As measures, the model includes different counts associated with employment, such as the number of active persons, employed persons, unemployed persons, registered contracts, job seekers, registered unemployment, and Social Security affiliates. 
edintemp: a qb:DataSet ; 
    qb:structure edintemp:dsd .

edintemp:dsd a qb:DataStructureDefinition ;
    qb:component
        [ qb:dimension edintdimension:refAdminRegion ; qb:order 1 ] ,
        [ qb:dimension edintdimension:sector ; qb:order 2 ] ,
        [ qb:dimension edintdimension:economicActivity ; qb:order 3 ] ,
        [ qb:dimension edintdimension:refPeriod ; qb:order 4 ] ,
        [ qb:dimension edintdimension:sex ; qb:order 5 ] ,
        [ qb:dimension edintdimension:age ; qb:order 6 ] ,
        [ qb:dimension edintdimension:contractType ; qb:order 7 ] ,
        [ qb:measure edintmeasure:activePopulationCount ; qb:order 8 ] ,
        [ qb:measure edintmeasure:employedPopulationCount ; qb:order 9 ] ,
        [ qb:measure edintmeasure:unemployedPopulationCount ; qb:order 10 ] ,
        [ qb:measure edintmeasure:registeredContractsCount ; qb:order 11 ] ,
        [ qb:measure edintmeasure:jobSeekersCount ; qb:order 12 ] ,
        [ qb:measure edintmeasure:registeredUnemploymentCount ; qb:order 13 ] ,
        [ qb:measure edintmeasure:socialSecurityAffiliatesCount ; qb:order 14 ] .

Example 1: Number of employed persons by area, period, sex, and age group back to index

This example represents the data for the number of employed persons in a given administrative region, for a specific reference period, broken down by sex and age group. Since this vocabulary already defines the dataset edintemp: and the data structure edintemp:dsd, there is no need to generate new dataset or structure instances; the existing ones can be reused.
The steps required to represent a specific observation are described below. **1) Reuse of the dataset and the data structure:** This vocabulary contains a single dataset, represented by edintemp:, which is linked to the structure edintemp:dsd through the property qb:structure. 
edintemp: a qb:DataSet ; 
    qb:structure edintemp:dsd .
**2) Identification of the relevant dimensions and measures:** Although the structure edintemp:dsd contains all the dimensions and measures of the employment cube, this example only uses the dimensions edintdimension:refAdminRegion, edintdimension:refPeriod, edintdimension:sex and edintdimension:age, together with the measure edintmeasure:employedPopulationCount. It is important to note that the dimension edintdimension:refPeriod has time:Interval as its range, so the reference period must be represented through an explicit instance of a temporal interval. 
<http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/interval/2025>
    a time:Interval ;
    rdfs:label "Year 2025"@en ,
               "Año 2025"@es .
**3) Definition of a dataset observation:** The final step consists of creating an instance of qb:Observation and linking it to the dataset through the property qb:dataSet. The corresponding values for the selected dimensions and measure are then assigned. 
<http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/observation/obs001>
    a qb:Observation ;
    qb:dataSet edintemp: ;
    edintdimension:refAdminRegion <http://vocab.linkeddata.es/datosabiertos/recurso/territorio/municipio/zaragoza> ;
    edintdimension:refPeriod <http://vocab.linkeddata.es/datosabiertos/recurso/actividad-economica/cubo-empleo/interval/2025> ;
    edintdimension:sex <http://purl.org/linked-data/sdmx/2009/code#sex-F> ;
    edintdimension:age "25-29"^^xsd:string ;
    edintmeasure:employedPopulationCount "12543"^^xsd:nonNegativeInteger .
This observation indicates that, in the municipality of Zaragoza, during the temporal interval corresponding to the year 2025, the number of employed women in the 25 to 29 years old age group is 12,543. This example shows how a single cube observation combines several dimension values with a specific measure to represent an employment indicator. In the same way, the same structure can be reused to represent other cube measures, such as the number of active persons, unemployed persons, registered contracts, job seekers, registered unemployment, or Social Security affiliates.
